# ISSUE-0002 数据引擎 MVP（参数采样与任务清单生成）

## 状态
- Done

## 背景
项目已完成流程与骨架搭建，需要进入技术路线第一阶段：数据引擎。  
首步应先实现“工况参数采样 + 任务清单生成”的 MVP，为后续批量 FDS 仿真提供输入。

## 目标
- 提供可复现的参数采样脚本。
- 生成统一格式任务清单文件（CSV）。
- 支持通过配置控制样本数量与取值范围。

## 验收标准
- 新增脚本可执行：`scripts/data/generate_cases.py`。
- 生成文件：`data/meta/cases_mvp.csv`。
- 输出字段至少包含：`case_id`、`fire_x`、`fire_y`、`hrr_peak_kw`、`vent_open_ratio`、`duration_s`。
- 可通过命令指定样本数（例如 100/1000）。

## 范围
- In Scope：
  - 规则采样（随机均匀）与固定随机种子。
  - CSV 输出与字段校验。
- Out of Scope：
  - FDS 实际运行与结果解析。
  - 训练数据预处理流水线。

## 风险
- 参数空间定义过窄导致后续泛化不足。
- 参数空间定义过宽导致样本效率低。

- [x] 实现采样脚本与 CSV 输出。
- [x] 编写使用说明与示例命令。
- [x] 运行一次自检并提交结果。
